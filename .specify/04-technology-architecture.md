# 04 — Technology Architecture (TOGAF ADM: Phase D)

## Technology Stack

| Layer | Technology | Version | Justification |
|---|---|---|---|
| **Framework** | Next.js (App Router) | 16.1.6 | SSR, routing, and API routes in a single project |
| **Language** | TypeScript | 5+ | Strict mode (`strict: true`) |
| **UI** | Tailwind CSS | 4 | Utility-first, fast iteration |
| **Components** | shadcn/ui | Latest | Professional components, no design effort |
| **Charts** | Recharts | 3.7.0 | Composable charts for dashboards |
| **ORM** | Prisma | 5.22.0 | Type-safe queries, schema management, seeder |
| **Database** | SQLite | — | Zero config, single file, portable |
| **Text AI** | Anthropic Claude (`claude-sonnet-4-20250514`) | SDK 0.78.0 | Best reasoning for text conversations and intent detection |
| **Voice AI** | OpenAI Realtime API (`gpt-4o-mini-realtime-preview`) | REST + WebRTC | Only option with native low-latency audio streaming for browser voice |
| **Audio transport** | WebRTC (browser native) | — | SDP exchange with OpenAI; no additional infrastructure |
| **Real-time** | Server-Sent Events (SSE) | — | Live updates for inbox and dashboard |
| **Validation** | Zod | 4.3.6 | Input validation on all API routes |
| **Date handling** | date-fns | 4.1.0 | SLA calculations and time formatting |
| **IDs** | uuid | 13.0.0 | Case + entity ID generation |
| **Containerization** | Docker | — | Railway deployment via Dockerfile |
| **Hosting** | Railway | — | Auto-deploy on push; supports SQLite persistence |
| **Runtime** | Node.js | 20+ | Required by Next.js + Prisma on Railway |

## Variables de entorno

```env
# Required
DATABASE_URL=file:./dev.db          # SQLite local path

# Optional — both have simulation fallbacks if absent
ANTHROPIC_API_KEY=sk-ant-...        # Text bot (Claude)

```

### Deployment (Production)

Railway auto-deploys on push to `claude/omnichannel-customer-ops-demo-w2Fu3` branch:

1. Railway builds the Docker image (`Dockerfile`)
2. `prisma generate && prisma db push` runs at build time
3. `postbuild` script runs seeder (`scripts/seed.js`) — generates 300 demo cases
4. `next start -p ${PORT}` starts the server

```json
// railway.json
{
  "build": { "builder": "DOCKERFILE", "dockerfilePath": "Dockerfile" },
  "deploy": { "restartPolicyType": "ON_FAILURE", "restartPolicyMaxRetries": 10 }
}
```

### Deployment (Local)

```bash
git checkout claude/omnichannel-customer-ops-demo-w2Fu3
npm install
cp .env.example .env.local    # fill in API keys
npm run dev
# → http://localhost:3000
```

## Security Architecture

- **Authentication:** None — single tenant, open access
- **Authorization:** None — all routes are public
- **Secrets management:** `.env.local` locally (gitignored); Railway env vars in production
- **API key exposure:** `ANTHROPIC_API_KEY` and `OPENAI_API_KEY` are server-side only — never exposed to the browser
- **HTTPS:** Enforced automatically by Railway in production
